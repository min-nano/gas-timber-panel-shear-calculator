「確定済みの前提」→「フェーズ別の作業」→「未決事項」の順にまとめています。

## 1. 確定済みのアーキテクチャと規約（新セッション向けの前提）

新しいセッションは、まず以下を守って着手すること。

### フォルダ構成
```
src/    … clasp のターゲット（.clasp.json の rootDir=src）。GAS 本体のみを置く。
tests/  … Node の node:test で回すユニットテスト。GAS API に依存しない純関数をテスト。
.github/workflows/ci.yml … テスト（全ブランチ/PR）＋ main への push で GAS 自動デプロイ。
```

### 計算ロジックは「唯一の実装（single source of truth）」
- 計算式は必ず `src/*.js` に**純関数**として実装し、ファイル末尾の環境ガード
  （`if (typeof module !== 'undefined' && module.exports) { ... }`）で Node からも読めるようにする。
  - GAS ではグローバル関数、Node テストでは `require()` で読み込む。
  - `SpreadsheetApp` 等の GAS 専用 API を計算関数に混ぜない（混ぜるとテスト不能になる）。
- UI（`src/index.html`）は `google.script.run` 経由でこのサーバ関数を呼ぶ。
  画面表示の数値が必ずテスト済みロジックと一致する状態を保つ。
- 既存 API: `computeNailArrayConstants(nails, panelArea)` を参考に、
  新しい計算も同じ粒度（式ごとの補助関数＋統合関数）で追加する。各関数に**グレー本の式番号**をコメントする。

### テスト方針
- 追加する各計算式に、①関数単位のユニットテスト、②**グレー本の解説にある計算例**を必ず含める。
- 実行は `npm test`（= `node --test tests/*.test.js`）。外部依存を増やさない。

### デプロイ / シークレット（clasp 3.x）
- CI は **@google/clasp 3.3.0** を使用（`package.json` で固定）。
- 認証: `CLASP_TOKEN`（`clasp login` 後の `~/.clasprc.json` の中身、`{ "tokens": {...} }` 形式）を
  そのまま `~/.clasprc.json` に書き出す。clasp 3.x は旧 V1 形式にも後方互換。
- `GAS_SCRIPT_ID` → `.clasp.json` に注入、`GAS_DEPLOYMENT_ID` → `clasp deploy -i` の対象。
- `.clasp.json` / `~/.clasprc.json` は `.gitignore` 済み（リポジトリに秘匿情報を置かない）。

### 実行権限
- `src/appsscript.json` は `executeAs: USER_ACCESSING`（アクセスユーザーとして実行）。
  各ユーザーが自分の権限で動くため、他人が勝手にスプレッドシートを作れない。
- Drive/Sheets を使うフェーズでは、**必要最小限の `oauthScopes` を `appsscript.json` に追記**する
  （例: `drive.file`, `spreadsheets`）。過剰なスコープは付けない。

### コーディング規約
- コメント・ドメイン用語は日本語、コード識別子は英語。周囲のスタイルに合わせる。
- 釘は「1 本 = 1 要素」で表現（座標の重複が本数の重みになる）。格子生成は `buildRectangularGrid`。

## 2. フェーズ別の作業計画

> 依存関係のゆるい順に並べています。フェーズは独立して着手可能ですが、
> **フェーズ3（3.3〜3.6 の耐力・剛性）はグレー本の該当ページ（式）の追加添付が前提**です（後述）。

### フェーズ1: マスタデータ（面材・釘・枠材）とプルダウン選択
- **目的**: 面材の種類・厚さ、釘（CN50 等）、枠材の樹種などをプルダウンで選び、パラメータを自動反映。入力漏れ・不整合を防ぐ。
- **スコープ**:
  - スプレッドシートに「マスタシート」を用意（面材／釘／枠材）。列例:
    - 面材: 種類, 厚さ, せん断弾性係数 等
    - 釘: 呼称（CN50…）, 一面せん断剛性, 一面せん断耐力 等
    - 枠材: 樹種, めり込み関連係数 等
  - GAS でマスタを読み込む API（`google.script.run`）を追加し、UI 起動時に取得してプルダウン化。
- **技術メモ**: マスタの所在（固定スプレッドシート ID をスクリプトプロパティで持つ／テンプレをコピー）を決める。読み取りは `spreadsheets.readonly` で足りる。
- **触るファイル**: `src/WebApp.js`（マスタ取得 API）, `src/index.html`（プルダウン）, `src/appsscript.json`（scope）。
- **完了条件**: マスタから選択 → 対応パラメータが UI に反映され、計算に使われる。

### フェーズ2: スプレッドシート連携（起動時に保存先指定 → リアルタイム保存 = 自動履歴）
- **目的**: 「保存ボタン」ではなく、起動時に**作業場所（Drive フォルダ）と保存スプレッドシート名**を決め、入力のたびに自動保存 → 履歴が自動的に残る。
- **スコープ**:
  - 起動時 UI: 保存先フォルダ／スプレッドシート名の指定（無ければ作成、あれば追記）。
  - 入力・計算のたびに debounce して行を upsert（タイムスタンプ付き）。追記型で履歴を蓄積。
- **技術メモ**:
  - scope は `drive.file`（アプリが作成/選択したファイルのみ）＋ `spreadsheets` を推奨（最小権限）。
  - `USER_ACCESSING` なので本人の Drive に保存され、他人が勝手に作らない要件を満たす。
  - 保存粒度（1 物件=1 シート／1 行=1 計算スナップショット等）と履歴の持ち方を決める。多重保存・競合に注意。
- **触るファイル**: `src/WebApp.js`（保存 API）, `src/index.html`（起動時ダイアログ, 自動保存）, `src/appsscript.json`（scope）。
- **完了条件**: 起動時に保存先を決めると、以降の変更が自動でシートに時系列で記録される。

### フェーズ3: 耐力壁・水平構面の剛性・耐力の詳細計算（グレー本 3.3〜3.6）
- **目的**: MVP の釘配列諸定数（Ixy/Zxy/Cxy）を用いて、面材張り耐力壁・水平構面の**せん断剛性・許容耐力・終局耐力**を算定。
- **前提（重要）**: 本リポジトリに取り込んだのは 3.2 節のみ。**3.3〜3.6 の該当ページ（式・記号・適用範囲）は未入手**。
  - → このフェーズの着手セッションで、グレー本 3.3〜3.6 の PDF を添付してもらう必要がある。
- **スコープ（想定）**: 釘の一面せん断剛性・耐力、面材のせん断剛性、めり込み、枠材変形、各種低減係数、適用範囲チェック。
- **技術メモ**: セクション1の規約どおり、式ごとの純関数＋統合関数で `src` に追加。各式に式番号コメント、解説の計算例をテスト化。
- **完了条件**: 実務で使う耐力・剛性が算定でき、解説の計算例と一致。適用範囲外は警告。

### フェーズ4: 計算書 PDF 出力（Print CSS / A4）
- **目的**: 画面はシンプルに保ちつつ、提出用の A4 計算書を PDF 出力。
- **スコープ**:
  - `@media print` で UI 部品を非表示、印刷用の「計算書フォーマット」（普段は `display:none`）を表示。
  - `@page { size: A4; margin: 15mm; }`、`window.print()` で仮想プリンタ→PDF。
  - 計算書内容: 物件情報ヘッダ、入力一覧、途中経過、最終結果、根拠（式番号）。
- **触るファイル**: 主に `src/index.html`（印刷用レイアウト＋CSS）。
- **完了条件**: 「PDF 出力」ボタンから、体裁の整った A4 計算書が得られる。

### 横断タスク（各フェーズと並行）
- 入力バリデーション強化、許容値超過などのエラー赤字表示（当初 UI 案）。
- 適用範囲（3.3〜3.6 の適用条件）を満たすかの自動チェックと警告。
- 単位表記の統一、UI/UX の微調整、物件情報（物件名・部位・日付・担当者）入力。

## 3. 未決事項（着手時に決める）

- **マスタの所在**: 固定スプレッドシート（スクリプトプロパティに ID）か、テンプレをコピーする方式か。
- **保存の粒度と履歴**: 物件単位／計算スナップショット単位、履歴の持ち方（追記行 or バージョン列）。
- **公開範囲**: `appsscript.json` の `access` を `ANYONE`（現状）にするか `DOMAIN`（社内限定）にするか。
- **グレー本 3.3〜3.6 の該当ページ**: フェーズ3 の前に PDF を用意する。
- **リアルタイム保存の頻度**: debounce 間隔、書き込み回数（API 制限）とのバランス。
